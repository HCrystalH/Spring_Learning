CHECK REPORT — LOGIN & DATA SEED FAILURE
=========================================

SYMPTOMS
---------
- App fails to start with ScriptStatementFailedException
- Login doesn't work because no instructor exists in DB

ROOT CAUSES FOUND & FIXED
--------------------------

1. [application.properties] Missing spring.sql.init.mode=always
   The default mode "embedded" may skip data.sql in some setups.
   → Added spring.sql.init.mode=always

2. [data.sql] MERGE INTO missing id column — NULL not allowed
   Tables use @GeneratedValue for id, but MERGE INTO without
   specifying id tries to insert NULL into a NOT NULL column.
   → Added explicit id values to all MERGE INTO statements

3. [data.sql] Column name mismatch: entity uses @Column(name="lookup_value")
   The Lookup entity maps field "value" to DB column "lookup_value",
   but the SQL used "value" which doesn't exist in the table.
   → Changed "value" to "lookup_value" in all MERGE INTO statements

4. [data/courseshare.mv.db] Stale database file
   Old DB was created before data.sql existed, so it had empty tables.
   → Deleted old files to force fresh database creation

CURRENT STATUS
---------------

✓ data.sql runs without errors on startup
✓ Seed data inserted: 1 instructor, 5 categories, 5 courses, 5 reviews, 5 lookups
